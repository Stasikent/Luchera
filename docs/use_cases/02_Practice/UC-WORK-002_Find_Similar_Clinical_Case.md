# UC-WORK-002

## Поиск похожего клинического случая

---

## Цель

Позволить специалисту быстро найти клинические случаи, максимально похожие на текущую диагностическую ситуацию, для сравнения, обучения и принятия обоснованного профессионального решения.

---

## Основные участники

- врач-рентгенолог;
- ординатор.

---

## Вторичные участники

- интеллектуальный помощник;
- поисковый сервис;
- база клинических случаев.

---

## Предусловия

- пользователь авторизован;
- в системе присутствует база клинических случаев;
- пользователю доступен модуль «Практика».

---

## Триггер

Во время анализа исследования специалист хочет найти похожие случаи.

Например:

- похожую рентгенологическую картину;
- похожее описание;
- аналогичное заключение;
- редкую патологию;
- похожий клинический вопрос.

---

## Основной сценарий

1. Пользователь открывает раздел поиска клинических случаев.
2. Вводит поисковый запрос.
3. При необходимости применяет фильтры.
4. Система выполняет поиск.
5. Отображается список наиболее релевантных случаев.
6. Пользователь открывает выбранный случай.
7. Система отображает:
   - изображения;
   - описание;
   - заключение;
   - обсуждение (при наличии);
   - диагноз;
   - дополнительные материалы;
   - источники.
8. Пользователь может перейти к связанным случаям.

---

## Альтернативные сценарии

### Поиск по диагнозу

Пользователь вводит диагноз.

Система показывает случаи с данным диагнозом.

---

### Поиск по описанию

Пользователь вводит часть описания.

Система ищет похожие описания.

---

### Поиск по заключению

Пользователь вводит предполагаемое заключение.

Система отображает похожие случаи.

---

### Поиск по локализации

Пользователь выбирает область исследования.

Система ограничивает поиск соответствующей анатомической областью.

---

## Исключительные ситуации

### Совпадения отсутствуют

Система сообщает об отсутствии похожих случаев и предлагает:

- изменить запрос;
- расширить критерии поиска;
- посмотреть похожие темы.

---

### Недостаточно данных

Система предлагает уточнить запрос.

---

## Результат

Пользователь получает один или несколько релевантных клинических случаев для анализа и сравнения.

---

---

## Ограничения

Сценарий не выполняет автоматическую постановку диагноза.

Система предоставляет материалы для анализа, а решение принимает специалист.

---

## Критерии успешности

- пользователь быстро находит релевантные случаи;
- результаты можно фильтровать;
- каждый случай содержит подтвержденный источник происхождения;
- случаи доступны для последующего изучения и обсуждения.
