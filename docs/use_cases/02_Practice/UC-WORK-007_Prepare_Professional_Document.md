# UC-WORK-007

## Подготовка профессионального документа

---

## Цель

Помочь специалисту быстро подготовить профессиональный документ на основании выбранного шаблона, нормативных требований и введённых пользователем данных.

---

## Рабочая задача

Подготовить документ, соответствующий требованиям профессиональной деятельности.

---

## Основные участники

- врач-рентгенолог;
- рентгенолаборант;
- руководитель подразделения.

---

## Вторичные участники

- интеллектуальный помощник;
- база шаблонов;
- база нормативных документов.

---

## Предусловия

- пользователь авторизован;
- доступны шаблоны документов;
- доступны нормативные материалы.

---

## Триггер

Пользователю необходимо подготовить профессиональный документ.

Например:

- информированное добровольное согласие;
- отказ от исследования;
- отказ от введения контрастного препарата;
- объяснительная записка;
- служебная записка;
- ответ пациенту;
- ответ администрации;
- проект локального документа;
- журнал;
- памятка пациенту.

---

## Основной сценарий

1. Пользователь выбирает тип документа.
2. Система предлагает подходящие шаблоны.
3. Пользователь выбирает шаблон.
4. Заполняет необходимые данные.
5. Система формирует проект документа.
6. Пользователь проверяет содержимое.
7. При необходимости редактирует документ.
8. Пользователь сохраняет или экспортирует документ.

---

## Альтернативные сценарии

### Создание документа по свободному запросу

Пользователь описывает необходимый документ естественным языком.

Система предлагает наиболее подходящий шаблон.

---

### Создание документа по рабочей ситуации

Пользователь открывает документ непосредственно из другого Use Case.

Например:

- отказ пациента;
- спорная ситуация;
- нарушение процедуры.

---

## Исключительные ситуации

### Шаблон отсутствует

Система предлагает наиболее похожий документ.

---

### Недостаточно данных

Система сообщает, какие сведения необходимо заполнить.

---

## Результат

Пользователь получает подготовленный проект документа, соответствующий выбранному шаблону и введённым данным.

---

## Ограничения

Система формирует проект документа.

Пользователь самостоятельно проверяет содержание документа перед использованием.

---

## Критерии успешности

- документ подготовлен за минимальное количество действий;
- соблюдена структура шаблона;
- при необходимости приведены ссылки на нормативные документы;
- пользователь может отредактировать документ перед использованием.

---
