# Use Cases

Данный раздел содержит подробные описания бизнес-процессов платформы «Лучера».

В отличие от Product Book, который описывает концепцию, архитектуру и основные принципы платформы, Use Cases определяют, каким образом пользователи взаимодействуют с системой для достижения конкретных профессиональных целей.

Каждый Use Case представляет собой законченный бизнес-процесс и является основой для проектирования пользовательского опыта, функциональных требований, архитектуры системы и последующей реализации.

---

# Назначение

Use Cases используются для:

- описания бизнес-процессов платформы;
- проектирования пользовательского опыта (UX);
- формирования функциональных требований;
- подготовки архитектурных решений;
- проектирования API;
- разработки программного обеспечения;
- подготовки тест-кейсов;
- определения состава MVP.

---

# Принципы

Use Cases строятся вокруг бизнес-процессов, а не вокруг пользовательских ролей.

Один бизнес-процесс может включать нескольких участников.

Каждый Use Case описывается в отдельном документе.

Каждый документ описывает только один законченный процесс.

---

# Структура раздела

## 01 Education

Образовательные процессы.

Описывает взаимодействие студентов, ординаторов, преподавателей и кафедры в процессе обучения.

---

## 02 Practice

Практическая деятельность.

Описывает использование платформы врачами-рентгенологами, рентгенолаборантами и заведующими отделениями при выполнении профессиональных обязанностей.

---

## 03 Professional Development

Профессиональное развитие.

Описывает процессы повышения квалификации, профессиональной переподготовки, подготовки к аккредитации и сопровождения образовательной траектории специалиста.

---

## 04 Career

Карьерное сопровождение.

Описывает процессы поиска вакансий, анализа рынка труда и формирования рекомендаций по профессиональному развитию.

---

## 05 Administration

Администрирование.

Описывает управление учреждениями, кафедрами, учебными группами, пользователями и ролями.

---

## 06 Platform

Платформенные сервисы.

Описывает регистрацию, авторизацию, управление профилем, поиск, уведомления, хранение файлов и другие сервисы, обеспечивающие работу платформы.

---

# Организация документов

Каждый раздел содержит:

- обзорный README;
- перечень Use Cases;
- отдельный документ для каждого Use Case.

Идентификаторы имеют следующий формат:

UC-EDU-001

UC-WORK-001

UC-DEV-001

UC-CAREER-001

UC-ADMIN-001

UC-SYS-001

---

# Связь с другими документами

Use Cases являются промежуточным уровнем между концепцией продукта и технической реализацией.

Каждый Use Case связан:

- с сущностями Domain Model;
- с функциональными модулями платформы;
- с Functional Requirements;
- с MVP;
- с архитектурными решениями (ADR);
- с пользовательским интерфейсом;
- с тест-кейсами.

Такая организация обеспечивает полную прослеживаемость проектных решений на протяжении всего жизненного цикла платформы.
