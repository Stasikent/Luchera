# UC-PLT-006

## Поиск по платформе

---

## Цель

Позволить пользователю выполнять единый поиск по всем доступным объектам платформы независимо от их типа.

---

## Рабочая задача

Найти необходимую информацию в платформе.

---

## Основные участники

- пользователь.

---

## Вторичные участники

- поисковый сервис;
- интеллектуальный помощник.

---

## Предусловия

- пользователь авторизован;
- индекс поиска содержит доступные пользователю объекты.

---

## Триггер

Пользователь выполняет поисковый запрос.

---

## Основной сценарий

1. Пользователь вводит поисковый запрос.
2. Система анализирует запрос.
3. Выполняется поиск среди доступных объектов.
4. Результаты группируются по типам.
5. Пользователь открывает необходимый объект.

---

## Альтернативные сценарии

### Поиск с фильтрами

Пользователь ограничивает область поиска.

---

### Интеллектуальный поиск

Система интерпретирует смысл запроса и предлагает наиболее релевантные результаты.

---

### Поиск внутри конкретного раздела

Пользователь ограничивает поиск одним модулем.

---

## Исключительные ситуации

### Результаты отсутствуют

Система предлагает похожие запросы.

---

### Недостаточно прав

Часть результатов скрывается.

---

## Результат

Пользователь получает доступ к найденным объектам платформы.

---

## Связанные сущности

- Поисковый запрос;
- Пользователь;
- Индекс поиска.

---

## Связанные модули

- Platform.

---

## Связанные Use Cases

- UC-KNOW-002 — Поиск материалов базы знаний.
- UC-WORK-001 — Поиск профессионального материала.
- UC-PD-003 — Поиск программы профессионального развития.

---

## Ограничения

Поиск выполняется только по объектам, доступным пользователю в соответствии с его правами доступа.

---

## Критерии успешности

- результаты релевантны запросу;
- поиск выполняется быстро;
- результаты сгруппированы по типам объектов.

---

## Возможности дальнейшего развития

- семантический поиск;
- поиск по медицинским изображениям;
- голосовой поиск;
- персонализация результатов;
- сохраненные поисковые запросы.
