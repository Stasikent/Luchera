# UC-PLT-001

## Регистрация пользователя

---

## Цель

Позволить новому пользователю создать учетную запись для работы с платформой «Лучера».

---

## Рабочая задача

Создать учетную запись пользователя.

---

## Основные участники

- пользователь.

---

## Вторичные участники

- платформа;
- сервис проверки контактных данных.

---

## Предусловия

- пользователь не авторизован;
- отсутствует учетная запись с указанными контактными данными.

---

## Триггер

Пользователь открывает страницу регистрации.

---

## Основной сценарий

1. Пользователь выбирает регистрацию.
2. Указывает необходимые данные.
3. Подтверждает согласие с условиями использования платформы.
4. Подтверждает контактные данные.
5. Система создает учетную запись.
6. Пользователь получает возможность войти в систему.

---

## Альтернативные сценарии

### Регистрация через внешнюю систему

Пользователь использует внешнего поставщика аутентификации.

---

### Регистрация по приглашению

Пользователь переходит по ссылке приглашения и завершает регистрацию.

---

## Исключительные ситуации

### Пользователь уже существует

Система предлагает выполнить вход или восстановить доступ.

---

### Контактные данные не подтверждены

Регистрация не завершается до успешного подтверждения.

---

## Результат

Создана новая учетная запись пользователя.

---

## Связанные сущности

- Пользователь;
- Учетная запись;
- Контактные данные.

---

## Связанные модули

- Platform.

---

## Связанные Use Cases

- UC-PLT-002 — Авторизация пользователя.
- UC-PLT-005 — Приглашение пользователя.

---

## Ограничения

Один пользователь не может зарегистрировать несколько учетных записей с одинаковыми контактными данными.

---

## Критерии успешности

- регистрация успешно завершена;
- контактные данные подтверждены;
- пользователь может выполнить вход.

---

## Возможности дальнейшего развития

- SSO;
- двухфакторная регистрация;
- корпоративная регистрация;
- автоматическое присоединение к организации.
