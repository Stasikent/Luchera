# Platform

Раздел описывает платформенные бизнес-процессы, обеспечивающие работу всей экосистемы «Лучера».

Платформенные процессы не решают самостоятельные профессиональные задачи пользователя. Они предоставляют общие сервисы, которые используются всеми функциональными модулями платформы.

Функциональность данного раздела используется независимо от роли пользователя и обеспечивает единый пользовательский опыт во всех разделах системы.

---

# Назначение

Раздел описывает процессы:

- регистрации пользователей;
- авторизации;
- управления профилем;
- управления организациями и принадлежностью пользователей;
- поиска информации;
- хранения файлов;
- уведомлений;
- интеллектуальной поддержки;
- настройки персонализации.

# Границы раздела

В раздел **Platform** входят:

- регистрация;
- авторизация;
- управление пользователями;
- профиль пользователя;
- управление организациями;
- поиск;
- уведомления;
- хранение файлов;
- интеграция с внешними системами;
- интеллектуальная поддержка.

В раздел не входят:

- обучение;
- профессиональная деятельность;
- правовая поддержка;
- карьерное сопровождение;
- профессиональное развитие;
- административные процессы конкретной организации.

Эти процессы описываются в соответствующих разделах Use Cases.

# Принципы

## Единый пользовательский опыт

Все платформенные сервисы должны работать одинаково независимо от используемого модуля.

## Безопасность

Все действия пользователей должны выполняться в соответствии с принятой моделью авторизации и разграничения прав доступа.

## Масштабируемость

Платформенные сервисы должны поддерживать развитие системы без изменения пользовательских сценариев.

## Независимость

Платформенные сервисы не должны зависеть от конкретного профессионального направления.

## Повторное использование

Платформенные сервисы реализуются один раз и используются всеми функциональными модулями платформы.
