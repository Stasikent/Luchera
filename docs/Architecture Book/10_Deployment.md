# Deployment

Deployment определяет принципы развертывания, эксплуатации и масштабирования платформы «Лучера».

Документ описывает логическую инфраструктуру системы и требования к окружениям разработки, тестирования и промышленной эксплуатации.

Конкретные конфигурационные файлы и сценарии развертывания находятся в репозитории проекта.

---

# Назначение

Архитектура развертывания должна обеспечивать:

- простое развертывание MVP;
- воспроизводимость окружений;
- масштабируемость;
- отказоустойчивость;
- безопасность;
- возможность автоматического обновления.

---

# Архитектурная схема

```text
                Internet
                    │
                    ▼
            Reverse Proxy
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
   Frontend                Backend API
                                │
       ┌────────────────────────┼────────────────────────┐

 PostgreSQL              Object Storage              Redis
                                │
                                ▼
                     Intelligence Services
```

Все внутренние сервисы взаимодействуют внутри защищенной сети.

---

# Окружения

Для платформы используются следующие окружения:

- Development;
- Testing;
- Staging;
- Production.

Каждое окружение должно быть максимально приближено к Production.

---

# Development

Используется разработчиками.

Особенности:

- локальное развертывание;
- упрощенная конфигурация;
- тестовые данные;
- возможность быстрой перезагрузки компонентов.

---

# Testing

Используется автоматическими тестами.

Особенности:

- полностью воспроизводимое окружение;
- автоматическое создание;
- автоматическое удаление после завершения тестирования.

---

# Staging

Используется перед выпуском новой версии.

Должен максимально соответствовать Production.

Используется для:

- приемочного тестирования;
- проверки миграций;
- проверки производительности;
- демонстрации новых возможностей.

---

# Production

Основное рабочее окружение платформы.

Особенности:

- отказоустойчивость;
- резервное копирование;
- централизованный мониторинг;
- журналирование;
- автоматическое восстановление сервисов.

---

# Контейнеризация

Все компоненты платформы поставляются в виде контейнеров.

Для MVP используется Docker Compose.

Архитектура должна поддерживать переход на Kubernetes без изменения бизнес-логики.

---

# Конфигурация

Конфигурация приложения хранится отдельно от исходного кода.

Используются:

- переменные окружения;
- файлы конфигурации;
- секреты инфраструктуры.

Конфиденциальные данные не должны храниться в репозитории.

---

# Масштабирование

Архитектура должна поддерживать независимое масштабирование:

- Backend;
- Frontend;
- Intelligence;
- Object Storage.

База данных масштабируется отдельными средствами PostgreSQL.

---

# Хранение данных

Постоянному хранению подлежат:

- база данных;
- пользовательские файлы;
- объекты базы знаний;
- резервные копии.

Временные данные не должны использовать постоянное хранилище без необходимости.

---

# Резервное копирование

Должно обеспечиваться:

- регулярное резервное копирование базы данных;
- резервное копирование файлов;
- проверка возможности восстановления;
- хранение нескольких поколений резервных копий.

---

# Мониторинг

Инфраструктура должна обеспечивать:

- мониторинг доступности сервисов;
- мониторинг производительности;
- сбор журналов;
- контроль ошибок;
- уведомления о критических событиях.

---

# Обновление

Обновление платформы должно выполняться с минимальным временем недоступности.

Перед выпуском новой версии обязательно выполняются:

- автоматические тесты;
- применение миграций;
- проверка работоспособности сервисов.

---

# Восстановление

При возникновении сбоя должна обеспечиваться возможность:

- восстановления базы данных;
- восстановления файлов;
- повторного развертывания сервисов;
- возврата к предыдущей версии системы.

---

# Связь с документацией

Deployment основан на:

- System Architecture;
- Technology Stack;
- Application Architecture.

Документ определяет принципы эксплуатации платформы и используется при настройке инфраструктуры.
