# Intelligence Architecture

Intelligence Architecture определяет архитектуру интеллектуальной поддержки платформы «Лучера».

Интеллектуальные сервисы являются частью платформы и используются всеми пользовательскими модулями.

Архитектура интеллектуальной поддержки не зависит от конкретного поставщика моделей искусственного интеллекта.

---

# Назначение

Интеллектуальная подсистема предназначена для:

- интеллектуального поиска;
- формирования рекомендаций;
- сопровождения обучения;
- сопровождения профессиональной деятельности;
- анализа профессиональных ситуаций;
- помощи пользователю при работе с платформой.

Интеллектуальная поддержка помогает пользователю принимать решения, но не принимает их вместо него.

---

# Общая архитектура

```text
                 User
                  │
                  ▼
          Intelligence API
                  │
      ┌───────────┼────────────┐
      ▼           ▼            ▼
 Context     Knowledge      User Profile
 Builder         │              │
      └──────────┼──────────────┘
                 ▼
            Prompt Builder
                 │
                 ▼
            LLM Provider
                 │
                 ▼
          Response Validator
                 │
                 ▼
          Intelligence Result
```

Интеллектуальная подсистема использует контекст пользователя и объекты базы знаний для формирования ответа.

---

# Основные компоненты

## Context Builder

Формирует профессиональный контекст пользователя.

Контекст может включать:

- профессиональную роль;
- текущий модуль;
- образовательный прогресс;
- профессиональную задачу;
- историю взаимодействия;
- используемые объекты базы знаний.

---

## Knowledge Access

Предоставляет интеллектуальным сервисам доступ к объектам базы знаний.

Все обращения к знаниям выполняются через публичные интерфейсы Knowledge.

Прямой доступ к базе данных запрещён.

---

## Prompt Builder

Формирует запрос к языковой модели.

Prompt Builder отвечает за:

- структуру запроса;
- выбор контекста;
- ограничение объёма;
- ссылки на источники.

---

## LLM Provider

Интерфейс взаимодействия с языковыми моделями.

Архитектура должна поддерживать возможность использования различных поставщиков.

Конкретная модель не должна влиять на бизнес-логику платформы.

---

## Response Validator

Проверяет ответ модели перед передачей пользователю.

Может выполнять:

- контроль формата;
- проверку ссылок;
- фильтрацию запрещённого содержимого;
- проверку полноты ответа.

---

# Retrieval-Augmented Generation (RAG)

При формировании ответов используется Retrieval-Augmented Generation.

Общий процесс:

```text
Вопрос пользователя

↓

Определение контекста

↓

Поиск по Knowledge

↓

Формирование Prompt

↓

LLM

↓

Проверка ответа

↓

Ответ пользователю
```

Модель должна использовать сведения из базы знаний платформы, а не только собственные внутренние знания.

---

# Источники информации

Приоритет источников:

1. Knowledge Asset.
2. Нормативные документы.
3. Клинические случаи.
4. Рабочие шаблоны.
5. История взаимодействия пользователя.

Ответы должны опираться на наиболее релевантные источники.

---

# Интеллектуальные рекомендации

Рекомендации формируются на основании:

- профессионального контекста;
- образовательной активности;
- компетенций;
- объектов базы знаний;
- профессиональных целей пользователя.

---

# Ограничения

Интеллектуальная подсистема не должна:

- принимать медицинские решения вместо пользователя;
- изменять пользовательские данные без подтверждения;
- скрывать источники информации;
- выдавать предположения за подтверждённые сведения.

---

# Масштабируемость

Архитектура должна поддерживать:

- смену поставщика LLM;
- использование нескольких моделей одновременно;
- локальные модели;
- специализированные медицинские модели.

Изменение поставщика моделей не должно требовать изменения бизнес-логики платформы.

---

# Безопасность

При использовании внешних языковых моделей:

- персональные данные должны быть защищены;
- медицинские изображения должны быть обезличены;
- передаваемый контекст должен быть минимально необходимым.

---

# Связь с документацией

Intelligence Architecture основана на:

- Product Book;
- Knowledge;
- Application Architecture;
- API Architecture.

Интеллектуальная подсистема использует общую доменную модель платформы и не изменяет её.
