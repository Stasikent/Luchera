# Architecture Book «Лучера»

> **Техническая архитектура единой цифровой экосистемы сопровождения специалистов лучевой диагностики.**

Architecture Book является продолжением Product Book.

Если Product Book отвечает на вопрос **«Что мы строим?»**, то Architecture Book отвечает на вопрос **«Как мы будем это строить?»**.

Документ определяет архитектурные принципы реализации платформы, организацию компонентов системы, взаимодействие модулей и технические решения, необходимые для разработки продукта.

Architecture Book предназначен для:

- архитекторов;
- backend-разработчиков;
- frontend-разработчиков;
- ML/AI-разработчиков;
- DevOps-инженеров;
- тестировщиков;
- новых участников команды.

---

# Связь с Product Book

Architecture Book не изменяет продуктовые требования.

Все технические решения принимаются на основании Product Book.

```text
Product Book
      │
      ▼
Architecture Book
      │
      ▼
Implementation
```

При изменении архитектуры не должны нарушаться:

- миссия продукта;
- доменная модель;
- пользовательские сценарии;
- функциональные требования.

---

# Статус

Architecture Book разрабатывается после завершения Product Book v1.0.

На текущем этапе определяются:

- архитектура системы;
- технология реализации;
- структура сервисов;
- модель данных;
- API;
- инфраструктура;
- безопасность;
- процесс разработки.

# Архитектурные принципы

При проектировании платформы используются следующие принципы:

- модульность;
- слабая связанность компонентов;
- единая доменная модель;
- расширяемость;
- независимость пользовательских модулей;
- повторное использование компонентов;
- безопасность по умолчанию;
- наблюдаемость;
- автоматизация процессов разработки.

---

# Основная идея

Архитектура платформы строится вокруг единой доменной модели и общей базы знаний.

Все пользовательские модули используют общие платформенные сервисы и взаимодействуют через единые интерфейсы.

```text
                Platform
                    │
                    ▼
               Knowledge
            ┌────┴────┐
            ▼         ▼
      Education   Practice
            └────┬────┘
                 ▼
    Professional Development
                 ▼
              Career
```

Техническая архитектура должна обеспечивать независимое развитие каждого модуля без нарушения целостности всей платформы.

# Документация проекта

Полная документация проекта включает:

- Product Book — описание продукта;
- Architecture Book — техническая архитектура;
- Decision Log — журнал продуктовых решений;
- ADR — журнал архитектурных решений;
- Glossary — единая терминология проекта;
- Style Guide — правила оформления документации;
- Use Cases — пользовательские сценарии;
- Functional Requirements — функциональные требования.

---

# Следующий этап

После завершения Architecture Book начинается проектирование пользовательских интерфейсов и реализация первой версии платформы.

Все изменения технической архитектуры должны сопровождаться обновлением соответствующих разделов Architecture Book и, при необходимости, оформлением нового ADR.

---

> **Architecture Book является единым источником технической архитектуры платформы «Лучера» и определяет принципы её реализации.**
