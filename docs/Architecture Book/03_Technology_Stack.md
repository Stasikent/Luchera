# Technology Stack

Technology Stack определяет основные технологии, используемые при разработке платформы «Лучера».

Выбор технологий выполняется исходя из требований Product Book, Architecture Principles и System Architecture.

При выборе технологий учитываются:

- простота сопровождения;
- масштабируемость;
- зрелость экосистемы;
- производительность;
- доступность специалистов;
- возможность постепенного развития платформы.

---

# Общая архитектура

Первая версия платформы реализуется как модульный монолит.

Используемые технологии должны поддерживать:

- единую кодовую базу;
- независимое развитие модулей;
- возможность последующего выделения сервисов;
- простое развертывание.

---

# Backend

Язык программирования:

- Python

Фреймворк:

- FastAPI

Основные причины выбора:

- высокая производительность;
- современная архитектура;
- автоматическая генерация OpenAPI;
- развитая экосистема;
- единый язык для backend и AI.

---

# Frontend

Язык:

- TypeScript

Фреймворк:

- React

Дополнительно:

- Vite
- React Router
- TanStack Query

Причины выбора:

- компонентная архитектура;
- высокая скорость разработки;
- зрелая экосистема;
- большое сообщество.

---

# База данных

Основная СУБД:

- PostgreSQL

Причины выбора:

- зрелость;
- надежность;
- поддержка сложных запросов;
- полнотекстовый поиск;
- расширяемость.

---

# ORM

- SQLAlchemy

Используется для:

- отображения доменной модели;
- миграций;
- работы с PostgreSQL.

---

# Миграции

- Alembic

---

# Объектное хранилище

Поддерживаются:

- S3;
- MinIO.

Используется для хранения:

- документов;
- изображений;
- презентаций;
- медицинских изображений;
- файлов пользователей.

---

# AI

Библиотеки:

- LangChain (при необходимости);
- LlamaIndex (при необходимости);
- OpenAI API;
- совместимые LLM API.

Интеллектуальная поддержка должна быть независимой от конкретной модели.

Замена поставщика LLM не должна изменять бизнес-логику платформы.

---

# Принципы выбора технологий

При изменении технологического стека должны соблюдаться следующие правила:

- доменная модель остается неизменной;
- Product Book не изменяется;
- публичные API сохраняются;
- архитектурные принципы продолжают соблюдаться.

Замена технологии должна требовать минимальных изменений бизнес-логики.
