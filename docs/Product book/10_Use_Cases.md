# Use Cases

Use Cases описывают основные бизнес-процессы, поддерживаемые платформой Лучера.

Они являются связующим звеном между концепцией продукта, функциональными требованиями и последующей реализацией.

Product Book содержит обзор структуры Use Cases.

Полные спецификации находятся в каталоге:

docs/use_cases/

---

# Назначение

Use Cases используются для:

- описания бизнес-процессов;
- проектирования пользовательского опыта;
- формирования функциональных требований;
- проектирования API;
- подготовки тест-кейсов;
- определения состава MVP.

---

# Принципы

Use Cases описывают:

- цель процесса;
- участников;
- ожидаемый результат.

Use Cases не описывают:

- пользовательский интерфейс;
- внутреннюю реализацию;
- архитектуру системы.

# Связь с другими документами

Use Cases являются основой для формирования функциональных требований.

Каждый Use Case может быть связан с одним или несколькими функциональными требованиями.

Каждое функциональное требование должно ссылаться минимум на один Use Case.
