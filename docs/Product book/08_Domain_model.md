# Доменная модель Лучеры

Доменная модель описывает основные объекты предметной области платформы «Лучера», их назначение и взаимосвязи.

Она формирует единое понимание продукта для разработчиков, медицинских специалистов, преподавателей, экспертов и других участников проекта.

Доменная модель не определяет структуру базы данных, программный код или способ реализации системы.

Её задача — определить язык предметной области, который используется во всей документации платформы.

---

# Принципы доменной модели

## Единый язык

Каждый объект имеет одно значение во всех разделах документации.

Одни и те же термины не должны использоваться для обозначения разных понятий.

---

## Независимость от реализации

Доменная модель описывает предметную область независимо от используемых технологий, пользовательского интерфейса или архитектуры программного обеспечения.

---

## Повторное использование

Каждый объект предметной области определяется только один раз и затем используется всеми подсистемами платформы.

---

## Расширяемость

Добавление новых направлений обучения, медицинских специальностей или профессиональных сервисов не должно требовать изменения существующей модели.

---

## Связанность

Основная ценность платформы определяется не отдельными объектами, а их взаимосвязями.

# Knowledge

Подсистема Knowledge содержит объекты профессиональных знаний, используемые всеми остальными модулями платформы.

Объекты базы знаний являются основным источником информации для обучения, профессиональной деятельности, карьерного сопровождения и интеллектуальной поддержки.

---

## Объект базы знаний (Knowledge Asset)

Базовый объект системы знаний.

Любая информация, предназначенная для хранения, поиска, повторного использования или анализа, представляется объектом базы знаний.

Объект базы знаний имеет собственный жизненный цикл, версионность, категории, теги, источники и связи с другими объектами.

---

## Тип объекта базы знаний (Knowledge Asset Type)

Определяет назначение объекта базы знаний.

Платформа не ограничивает перечень возможных типов.

Примеры:

- статья;
- клинический случай;
- нормативный документ;
- рабочий шаблон;
- презентация;
- изображение;
- видео;
- памятка;
- чек-лист;
- алгоритм;
- профессиональная рекомендация;
- другой тип.

Новые типы могут добавляться без изменения доменной модели.

---

## Категория

Используется для построения постоянной иерархической структуры базы знаний.

Категории формируют навигацию по профессиональным областям.

Каждый объект базы знаний может принадлежать одной или нескольким категориям.

---

## Тег

Тематическая метка объекта базы знаний.

Теги используются для:

- поиска;
- фильтрации;
- интеллектуальных рекомендаций;
- формирования тематических подборок.

Теги не образуют иерархию.

---

## Источник

Первоисточник информации, на основании которого создан объект базы знаний.

Источник может содержать:

- наименование;
- ссылку;
- автора;
- дату публикации;
- редакцию;
- сведения об актуальности.

Значимые профессиональные материалы должны иметь ссылки на первичные источники.

# Education

Подсистема Education использует объекты базы знаний для формирования образовательного процесса.

---

## Образовательная программа

Структурированный набор объектов базы знаний, предназначенный для достижения образовательной цели.

Программа определяет последовательность изучения материалов, выполнения заданий и прохождения контроля знаний.

---

## Учебный модуль

Логически завершенная часть образовательной программы.

Учебный модуль объединяет объекты базы знаний по определенной теме или компетенции.

---

## Задание

Практическая или теоретическая деятельность, которую должен выполнить обучающийся.

Задание может использовать объекты базы знаний, клинические случаи или профессиональные документы.

---

## Проверка знаний (Assessment)

Процесс оценки результатов обучения.

Проверка знаний может выполняться с использованием тестов, клинических случаев, практических заданий и других образовательных инструментов.

---

## Попытка прохождения

Зафиксированное выполнение задания, теста или клинического случая конкретным пользователем.

Попытка содержит результаты выполнения, затраченное время, полученные подсказки и итоговую оценку.

---

## Прогресс обучения

Совокупность сведений о прохождении образовательной программы.

Используется для отображения достигнутых результатов, определения степени завершения программы и формирования рекомендаций по дальнейшему обучению.

# Practice

Подсистема Practice использует объекты базы знаний для поддержки повседневной профессиональной деятельности специалиста.

Practice не заменяет медицинскую информационную систему или PACS. Ее задача — предоставить специалисту необходимые знания, рекомендации и профессиональные сервисы в процессе работы.

---

## Профессиональная задача

Профессиональная задача представляет собой рабочую ситуацию, при выполнении которой специалист использует возможности платформы.

Примеры профессиональных задач:

- интерпретация исследования;
- поиск нормативной информации;
- подготовка профессионального документа;
- анализ спорной ситуации;
- контроль качества исследования;
- проверка оснащения кабинета.

Профессиональная задача определяет контекст использования объектов базы знаний.

---

## Клинический случай

Тип объекта базы знаний, содержащий структурированное описание диагностической ситуации.

Клинический случай используется для:

- обучения;
- профессионального анализа;
- поиска аналогичных случаев;
- формирования рекомендаций.

Клинический случай может быть связан с другими объектами базы знаний.

---

## Медицинское изображение

Изображение, связанное с объектом базы знаний.

Медицинское изображение может использоваться:

- в клиническом случае;
- в образовательной программе;
- в профессиональном материале;
- в профессиональном документе.

Поддерживаемые типы изображений определяются возможностями платформы.

---

## Профессиональный документ

Документ, формируемый или используемый специалистом при выполнении профессиональной деятельности.

Примеры:

- описание исследования;
- заключение;
- протокол;
- отчет;
- служебная записка;
- экспертное заключение.

Профессиональный документ может использовать рабочие шаблоны и связываться с объектами базы знаний.

---

## Рабочий шаблон

Тип объекта базы знаний, предназначенный для подготовки профессиональных документов.

Рабочий шаблон определяет рекомендуемую структуру документа и может содержать обязательные разделы, подсказки и ссылки на связанные материалы.

---

## Профессиональная ситуация

Описание профессиональной ситуации, требующей анализа, принятия решения или применения нормативных требований.

Примеры:

- исследование беременной пациентки;
- отсутствие документов;
- конфликтная ситуация;
- необходимость дополнительного исследования;
- контроль качества.

Профессиональная ситуация может быть связана с нормативными документами, клиническими случаями и профессиональными рекомендациями.

# Professional Development

Подсистема Professional Development объединяет результаты обучения и профессиональной деятельности для планирования дальнейшего развития специалиста.

---

## Компетенция

Профессиональное знание, умение или навык, необходимый специалисту.

Компетенции используются для:

- формирования образовательных программ;
- оценки профессионального уровня;
- планирования развития;
- карьерного сопровождения.

Компетенции являются связующим элементом между обучением, практикой и карьерой.

---

## План профессионального развития

Персональный план развития специалиста.

План определяет последовательность образовательных программ, профессиональных активностей и целей развития.

---

## Образовательная активность

Подтвержденное участие пользователя в образовательном мероприятии.

Образовательная активность может быть получена внутри платформы или импортирована из внешних систем.

---

## Аккредитация

Профессиональное событие, требующее выполнения установленных требований для допуска к профессиональной деятельности.

Аккредитация связана с образовательными активностями, компетенциями и профессиональным развитием пользователя.

---

## Профессиональное портфолио

Структурированное представление профессиональных достижений пользователя.

Портфолио может включать:

- образование;
- компетенции;
- образовательные активности;
- сертификаты;
- аккредитации;
- публикации;
- достижения;
- профессиональный опыт.

Профессиональное портфолио используется для формирования карьерного профиля.

# Career

Подсистема Career использует сведения о профессиональном развитии специалиста для сопровождения его карьерного роста.

Подсистема помогает пользователю определять карьерные цели, находить профессиональные возможности и планировать дальнейшее развитие.

---

## Карьерная цель

Желаемый профессиональный результат, которого стремится достичь специалист.

Карьерная цель определяет направление профессионального развития и может быть связана с требуемыми компетенциями, образовательными программами и карьерными возможностями.

Примеры:

- врач-рентгенолог экспертного уровня;
- заведующий отделением;
- преподаватель кафедры;
- специалист по МРТ;
- научная деятельность;
- международная практика.

---

## Карьерная возможность

Возможность профессионального развития, соответствующая карьерным интересам пользователя.

Карьерная возможность может быть представлена в различных формах:

- вакансия;
- стажировка;
- образовательная программа;
- научный проект;
- преподавательская деятельность;
- грант;
- конкурс;
- программа обмена.

Карьерная возможность содержит сведения о требованиях, условиях участия и ожидаемых компетенциях.

---

## Карьерный профиль

Представление профессионального опыта пользователя, предназначенное для использования в карьерных процессах.

Карьерный профиль формируется на основе профессионального портфолио, но может содержать только выбранные пользователем сведения.

Пользователь может поддерживать несколько карьерных профилей для различных профессиональных направлений.

---

## Карьерный путь

Последовательность этапов, позволяющих достичь выбранной карьерной цели.

Карьерный путь может включать:

- развитие компетенций;
- образовательные программы;
- профессиональную деятельность;
- образовательные активности;
- аккредитацию;
- получение профессионального опыта.

Карьерный путь является персональным и может изменяться в зависимости от профессионального развития пользователя.

# Intelligence

Подсистема Intelligence обеспечивает интеллектуальную поддержку пользователя во всех модулях платформы.

Интеллектуальная поддержка не является самостоятельным профессиональным модулем и используется другими подсистемами платформы.

---

## Интеллектуальная рекомендация

Рекомендация, сформированная платформой на основании профессионального контекста пользователя.

Рекомендации могут использоваться:

- в обучении;
- в профессиональной деятельности;
- при профессиональном развитии;
- при карьерном сопровождении.

---

## Подсказка

Дозированная интеллектуальная помощь пользователю при выполнении профессиональной задачи.

Подсказка должна помогать пользователю самостоятельно принять решение и не заменять профессиональное мышление.

---

## Диалог

Последовательность взаимодействий пользователя с интеллектуальной системой.

Диалог может использоваться:

- для поиска информации;
- анализа профессиональной ситуации;
- получения рекомендаций;
- сопровождения обучения.

---

## Профессиональный контекст

Совокупность сведений, используемых платформой для формирования интеллектуальной поддержки.

Профессиональный контекст может включать:

- профессиональную роль;
- текущую задачу;
- используемые объекты базы знаний;
- компетенции;
- историю взаимодействия;
- образовательный прогресс;
- профессиональный опыт.

Профессиональный контекст не является самостоятельным модулем платформы, но используется всеми сервисами интеллектуальной поддержки.

# Связи между объектами

Платформа построена вокруг единой базы знаний.

Все остальные объекты используют знания для решения собственных задач.

```text
Knowledge Asset
        │
        ├──────────────┐
        │              │
        ▼              ▼
Education        Practice
        │              │
        └──────┬───────┘
               ▼
Professional Development
               ▼
Career
```

Основные связи:

- объект базы знаний используется образовательными программами;
- объект базы знаний используется профессиональной деятельностью;
- образовательная программа способствует развитию компетенций;
- профессиональная деятельность также способствует развитию компетенций;
- компетенции используются при формировании профессионального портфолио;
- профессиональное портфолио является основой карьерного профиля;
- карьерный профиль используется при поиске карьерных возможностей;
- карьерная цель определяет карьерный путь пользователя;
- интеллектуальная поддержка использует профессиональный контекст и объекты базы знаний во всех подсистемах платформы.
