# MVP

## Назначение

Минимально жизнеспособная версия (Minimum Viable Product, MVP) предназначена для проверки ключевых продуктовых гипотез платформы «Лучера» в условиях реальной эксплуатации.

Первая версия должна подтвердить, что единая цифровая экосистема действительно помогает специалистам лучевой диагностики эффективнее обучаться, использовать профессиональные знания и взаимодействовать с преподавателями.

MVP представляет собой законченную версию продукта, решающую ограниченный круг задач, достаточный для проведения пилотного внедрения.

---

# Цели MVP

Первая версия платформы должна подтвердить следующие гипотезы:

- пользователи готовы использовать единую профессиональную платформу вместо нескольких разрозненных сервисов;
- единая база знаний востребована на протяжении всего образовательного процесса;
- клинические случаи являются эффективным инструментом обучения;
- преподаватели готовы сопровождать образовательный процесс внутри платформы;
- интеллектуальные сервисы помогают быстрее находить необходимую профессиональную информацию.

---

# Пилотное внедрение

Первая версия платформы предназначена для пилотного внедрения на кафедре лучевой диагностики.

Участниками пилота являются:

- преподаватели;
- студенты;
- ординаторы.

При необходимости к пилотному проекту могут подключаться практикующие специалисты для оценки профессиональных сервисов.

---

# Основные пользовательские сценарии

После завершения MVP пользователь должен иметь возможность:

- зарегистрироваться;
- присоединиться к образовательной организации;
- получить доступ к образовательной программе;
- изучить учебные материалы;
- решить клинический случай;
- пройти тестирование;
- получить результаты обучения;
- найти профессиональную информацию;
- воспользоваться интеллектуальной поддержкой;
- продолжить обучение с сохранением прогресса.

Преподаватель должен иметь возможность:

- создавать образовательные программы;
- формировать задания;
- использовать клинические случаи;
- отслеживать прогресс обучающихся;
- анализировать результаты обучения.

---

# Состав MVP

## Platform

В первую версию входят:

- регистрация пользователей;
- авторизация;
- профиль пользователя;
- роли;
- организации;
- учебные группы;
- глобальный поиск;
- уведомления.

---

## Knowledge

В первую версию входят:

- объекты базы знаний;
- категории;
- теги;
- источники;
- поиск;
- просмотр материалов.

---

## Education

В первую версию входят:

- образовательные программы;
- учебные модули;
- задания;
- тестирование;
- клинические случаи;
- прогресс обучения.

---

## Practice

В MVP реализуются базовые профессиональные сервисы:

- поиск профессиональной информации;
- поиск похожих клинических случаев;
- поиск нормативной информации;
- получение профессиональной помощи.

Основная задача — оценить востребованность сервисов профессиональной поддержки.

---

## Professional Development

В первую версию входят:

- базовые компетенции;
- образовательная активность.

Карьерное сопровождение в MVP не реализуется.

---

# Что не входит в MVP

Первая версия не включает:

- полноценный модуль Career;
- профессиональное портфолио;
- автоматическую генерацию документов;
- расширенную аналитику;
- интеграцию с медицинскими информационными системами;
- импорт медицинских изображений;
- мобильные приложения;
- публичный API;
- офлайн-режим.

Эти возможности рассматриваются в следующих этапах развития платформы.

---

# Критерии успешности

Пилотное внедрение считается успешным, если подтверждаются следующие показатели:

- преподаватели используют платформу в образовательном процессе;
- студенты регулярно проходят образовательные программы;
- пользователи активно используют базу знаний;
- клинические случаи используются в обучении;
- интеллектуальный поиск востребован пользователями;
- пользователи положительно оценивают удобство платформы.

Результаты пилота используются при планировании дальнейшего развития продукта.

---

# Следующий этап

После завершения пилотного внедрения планируется развитие следующих направлений:

- расширение возможностей Practice;
- внедрение Professional Development в полном объеме;
- запуск Career;
- развитие интеллектуальных сервисов;
- интеграция с внешними информационными системами;
- расширение базы знаний;
- масштабирование платформы на другие медицинские специальности.
